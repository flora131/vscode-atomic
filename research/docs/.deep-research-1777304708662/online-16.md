The `extensions/php-language-features/` partition relies entirely on the VS Code extension API (`vscode` module) and Node.js standard library (`child_process`, `string_decoder`, `path`) with no external third-party libraries requiring documentation lookup. The one non-obvious dependency is the `which` npm package used in `validationProvider.ts` to locate the `php` binary on PATH, but this is a trivial utility with no porting complexity beyond replacing it with Rust's equivalent `which` crate or a direct `std::process::Command` PATH search. All language intelligence (completion, hover, signature help) is implemented as static lookup tables over hand-curated PHP global function and variable data embedded in `phpGlobalFunctions.ts` and `phpGlobals.ts`, making this partition self-contained and straightforward to port: the VS Code language-provider registration APIs map directly to LSP server capabilities in a Tauri/Rust context, and the `child_process.spawn` call for invoking the PHP binary maps directly to `std::process::Command`.
