**Justification:** The three runtime dependencies declared in `extensions/configuration-editing/package.json` — `@octokit/rest`, `jsonc-parser`, and `tunnel` — are TypeScript/Node.js libraries that exist entirely within the VS Code extension host process and are not being ported to Rust/Tauri. In a Tauri port, the settings-editing IntelliSense feature would be re-implemented using VS Code's built-in `vscode.languages` JSON schema registration API (which remains unchanged on the TypeScript/webview side), the GitHub Gist profile-export feature would call the GitHub REST API directly via Rust's `reqwest` crate or a Tauri HTTP command (making `@octokit/rest` irrelevant), JSON-with-comments parsing is handled by the VS Code language server infrastructure rather than any exposed Rust surface, and the `tunnel` library's sole job of constructing an HTTP-proxy `https.Agent` for Node's TLS stack has no Rust/Tauri counterpart because Tauri's native HTTP client handles proxy configuration through OS settings or Rust environment variables. Because none of these three libraries have a Rust API surface, a Tauri migration path, or architectural decisions that external documentation would illuminate, no external library documentation is central to this partition.
