The `extensions/yaml/` directory contains no TypeScript source files, no Electron APIs, and no runtime code of any kind — it consists entirely of declarative JSON artifacts (TextMate grammar files, a `language-configuration.json`, a `package.json` contribution manifest, and an NLS strings file) plus a single build-time Node script (`build/update-grammar.js`) that uses `vscode-grammar-updater` to pull updated grammar files from the upstream `RedCMD/YAML-Syntax-Highlighter` repository; because the port target is Tauri/Rust and none of these assets have any Electron or TypeScript dependency, the entire extension can be carried over as-is — the `.tmLanguage.json` grammars are consumed by whatever syntax-highlighting engine (e.g., tree-sitter or a TextMate-compatible tokenizer) the Tauri shell exposes, the `language-configuration.json` bracket/comment rules are pure data, and the `vscode-grammar-updater` script remains a development-time utility that runs under Node regardless of the host IDE.
