The `extensions/git/` partition has no runtime library that requires external documentation for porting assessment. The package.json lists five runtime dependencies — `byline` (line-splitting over Node streams), `picomatch` (glob matching), `which` (PATH executable lookup), `file-type` (MIME detection from buffer), and `vscode-uri` (URI normalization) — all of which are thin, well-understood utilities with direct Rust crate equivalents (`tokio`/`BufReader` lines, `globset`/`ignore`, `which`, `infer`, and `url` respectively) that require no dedicated research. The actual SCM logic uses `child_process.spawn` to invoke the system `git` binary and streams its stdout through `byline`; this design maps naturally to Rust's `std::process::Command` or `tokio::process::Command` without involving any complex third-party library whose API or behaviour would be non-obvious to a porter.
