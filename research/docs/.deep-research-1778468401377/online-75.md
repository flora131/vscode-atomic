The `extensions/latex/` scope consists entirely of static, declarative TextMate grammar JSON files (`TeX.tmLanguage.json`, `LaTeX.tmLanguage.json`, `Bibtex.tmLanguage.json`, `cpp-grammar-bailout.tmLanguage.json`, `markdown-latex-combined.tmLanguage.json`), language configuration JSON files, and a single build-time Node.js script (`build/update-grammars.js`) that pulls upstream grammar snapshots from the `jlelong/vscode-latex-basics` repository via the `vscode-grammar-updater` utility. None of these artifacts contain TypeScript, Electron APIs, or any runtime host coupling — the grammars are pure data consumed by whatever TextMate-compatible tokenisation engine the host provides, and the updater script runs only at build time, not at runtime. Porting the Tauri/Rust host therefore has zero impact on this extension: the JSON grammar files are copied verbatim, the language-configuration files are identical, and the grammar-updater script continues to run as a standalone Node.js build step regardless of the underlying IDE runtime.
