### Cross-Cutting Synthesis

This partition is a **pure TextMate grammar extension**. Its sole implementation file (`build/update-grammars.js`) is a 14-line build-time utility that downloads pre-generated grammar JSON from two upstream GitHub repositories and writes them to `./syntaxes/`. At runtime, VS Code's TextMate tokenization engine reads the two `.tmLanguage.json` artefacts directly; no TypeScript or Rust code from this extension is loaded. The extension contributes nothing beyond declarative JSON manifests and the two grammar files.

**Why this partition is non-essential to a Tauri/Rust port**

The `build/update-grammars.js` script is a development-time artefact maintenance tool. The runtime deliverables it produces — `syntaxes/objective-c.tmLanguage.json` and `syntaxes/objective-c++.tmLanguage.json` — are static JSON grammar files in TextMate format. Any editor capable of consuming TextMate grammars (including Tauri-based editors that integrate a grammar engine such as `tree-sitter` or `syntect`) can consume these files verbatim. The grammar files themselves require no TypeScript/Electron APIs, no VS Code extension host, and no Node.js runtime. Porting this partition to Tauri/Rust reduces to: (1) copying the two `.tmLanguage.json` files, and (2) replacing the `update-grammars.js` download script with an equivalent shell script, Makefile rule, or Rust build script — a trivial one-to-one substitution. No architectural decisions depend on this partition.
